# ADR-0037: Bevy 0.19 移行で採否した新機能

## Status

Accepted (2026-06-28、Issue #9 で適用)

## Context

Bevy 0.19.0 がリリースされた (https://bevy.org/news/bevy-0-19/)。当 repo は 0.18 採用が
engine 立ち上げ当初の選択で、まだ依存が浅いうちに追従しておく方針 (Issue #9)。

0.19 で増えた / 変わった項目のうち、当 repo として **採否を決めておきたい** ものを以下に並べる:

| 項目 | 概要 |
|---|---|
| BSN (Bevy Scene Notation) | 新しい declarative scene 構文。`scenes/*.rs` (本 repo の `*ScenePlugin`) との関係整理が要る。 |
| Resources-as-Components (内部実装) | `Resource` が `Component` も実装する内部変更。`Res<T>` の使用方法は据え置き。 |
| Render Graph → Schedules | カスタムレンダリング系の刷新。当 repo は未使用。 |
| `DiagnosticsOverlayPlugin` | 旧 `FpsOverlayPlugin` の後継候補。FPS / 各種指標のオーバーレイ表示。 |
| App Settings | 新規構造化設定機構。 |
| `bevy-inspector-egui` 0.19 対応版 | 外部 crate。本 repo は未使用だが将来の dev tool 候補。 |
| Parley text overhaul + `FontSize` enum | `font_size: f32` → `font_size: FontSize` (= `FontSize::Px(...)` / `Vw` / `Vh` / `VMin` / `VMax` / `Rem`)。0.19 で唯一の compile-breaking change として当 repo に届いた。 |
| subsecond hotpatching | 0.17 で本体統合 → 0.18 継続。0.19 リリースノートでの言及なし。本 repo の engine は dx serve hot reload を使っていない (editor 側のみ)。 |
| audio feature (`wav` 等) | 0.19 で audio feature flag が再編 (`wav` / `mp3` / `flac` 等が explicit) という流れ。`features = ["wav"]` 指定は 0.19 でもそのまま通る。 |

## Decision

### 本 MR で採用

**なし**。バージョン bump (`packages/engine/Cargo.toml` の `bevy = "0.18"` → `"0.19"`) と、
以下 2 系統の追随のみ:

- compile-breaking な `FontSize` enum 対応 (9 箇所、`scenes/title.rs` / `scenes/options.rs` /
  `features/character/state_debug.rs` で `font_size: <f32>` → `font_size: FontSize::Px(<f32>)`)。
  Parley text overhaul で `TextFont.font_size` の型が `f32` → `FontSize` enum (= `Px` / `Vw` /
  `Vh` / `VMin` / `VMax` / `Rem`) に変わったため。本 repo の用途は全部 window pixel ベースなので
  `FontSize::Px(...)` に一様に倒す。
- smoke test (`packages/engine/tests/engine_smoke.rs`) で `MeshPlugin` を追加。Bevy 0.19 では
  system param validation が build time → fetch time にずれ、GizmoPlugin 内の
  `draw_skinned_mesh_bounds` system が `Res<Assets<Mesh>>` と
  `Res<Assets<SkinnedMeshInverseBindposes>>` の不在で fetch-time panic するようになった。
  両 asset を一括 init する `bevy::mesh::MeshPlugin` を smoke test の Plugin 列に追加し、
  以後 gizmo 系の新規 asset 要件にも自動追従できるようにする。

### 採用しない

- **BSN (Bevy Scene Notation)** — 既存 `scenes/*.rs` の `SceneState` enum + `*ScenePlugin`
  構造 (`title` / `options` / `battle` / `result`) は Phase 3 まで運用しても破綻していない。
  BSN への置き換えは scene 構築の declarative 化と引き換えに「外部 `.bsn` ファイル管理 + asset
  hot reload 経路の追加学習」というコストを払う必要があり、現状の Plugin ベースで十分機能して
  いる以上、移行のメリットがコストを上回らない。将来 scene 数が二桁を超えて宣言的記述の
  価値が出てきたら再評価する。

### 別 Issue で採用評価

- **DiagnosticsOverlayPlugin** — F2 で出る `state_debug` (`AI=<state>` / `HP=...` 等) と
  併用したいかは未確定。FPS 表示は dev only で別レイヤ管理する選択肢もあり、別 Issue で
  既存 `state_debug` との配置 / レイヤ衝突を含めて評価する。
- **App Settings** — `bebeu-engine.yml` (ADR-0016) との二段構成にどう乗せるかが本題。
  別 Issue で扱う。
- **`bevy-inspector-egui` 0.19 対応版** — crates.io で 0.19 対応版が公開されたら別 Issue で
  evaluate。dev tool として `--features dev_inspector` ゲートで入れる選択肢がありえる。

### 確認結果のみ残す項目

- **subsecond hotpatching** — リリースノートに撤去言及なし、APIs/Cargo features での
  bring-up が必要な構成。本 repo の engine は dev で使っていない (editor は dx serve)。
  → 0.19 でも継続している前提で問題なし、影響なし。
- **`wav` feature** — `cargo build` で 0.19 でもそのまま通る。実機起動
  (`just engine-run-sample`) で audio スタックが panic せず立ち上がるところまで確認
  (battle 進入後の SE 発火目視は別途必要、本 MR の検証スコープ外)。

## Consequences

- `Cargo.toml` の `bevy = "0.19"` を起点に、以降の features 追加 / breaking changes は
  本 ADR 系列の補追 or 別 ADR で判断する。
- 採用見送りした BSN / Overlay / Inspector / App Settings は各々別 Issue で扱う。本 MR の
  scope は「移行を完了させる」ところまで。
- 0.19 移行に伴う API path 変更は **Parley の `FontSize` enum 化以外は届かなかった**。
  これは Issue #9 の事前 grep (`FpsOverlayPlugin` / `ScreenshotManager` / `RenderGraph` の
  使用 0 件) と整合する結果で、立ち上げ当初の依存浅さが効いている。今後 ADR-0035 系の
  AI 拡張や ADR-0026 系の render target 経路が増えるほど移行コストは上がるため、
  Bevy のメジャー追従は早めに回す方針を継続する。
